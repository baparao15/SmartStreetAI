## SECTION 2: AGENT SPECIFICATIONS

### 2.1 Data Ingestion Agent

**Purpose:** Continuous data collection and preprocessing

**Data Sources to Integrate:**
| Source | Data Type | Frequency | Priority |
|--------|-----------|-----------|----------|
| NSE Bhavcopy | EOD prices, volumes | Daily | Critical |
| NSE Corporate Filings | Results, announcements, board meetings | Real-time | Critical |
| BSE Announcements | Bulk deals, insider trading | Real-time | High |
| SEBI Filings | IPO updates, regulatory changes | Hourly | High |
| Economic Times API | News, analysis, expert views | Continuous | Medium |
| Social Sentiment | Twitter/Reddit retail sentiment | Continuous | Low |

**Processing Pipeline:**
1. **Fetch** → Pull from source APIs/feeds
2. **Validate** → Schema check, data quality flags
3. **Enrich** → Add metadata (sector, market cap, historical context)
4. **Store** → Time-series DB for prices, document store for filings
5. **Event Detection** → Identify significant changes vs. historical baseline

**Key Events to Flag:**
- Earnings surprises (>10% vs. consensus)
- Unusual volume spikes (>3x 20-day average)
- Price gaps (>5% overnight)
- Bulk deals (>1% of market cap)
- Insider buying/selling clusters
- Management commentary sentiment shifts
- Regulatory filing anomalies

---

### 2.2 Pattern Recognition Agent

**Purpose:** Detect technical patterns across NSE universe

**Pattern Library:**

| Pattern Category | Specific Patterns | Detection Method | Confidence Scoring |
|------------------|-------------------|------------------|-------------------|
| **Breakouts** | Resistance breakout, Channel breakout, Triangle breakout | Price level breach + volume confirmation | Volume ratio × price momentum |
| **Reversals** | Head & Shoulders, Double Top/Bottom, Divergence | Pivot point analysis + indicator divergence | Pattern completion % + volume profile |
| **Continuation** | Flag, Pennant, Cup & Handle | Consolidation detection + measured move | Time in pattern + volume contraction |
| **Support/Resistance** | Key levels, Fibonacci retracements | Historical touch points clustering | Number of touches + recency |
| **Candlestick** | Doji, Engulfing, Morning Star | Pattern matching algorithm | Body/wick ratios + context |

**Technical Indicators to Calculate:**
- Moving averages (20, 50, 200 EMA/SMA)
- RSI (14-period)
- MACD (12,26,9)
- Bollinger Bands (20,2)
- Volume-weighted average price (VWAP)
- Average True Range (ATR) for volatility

**Pattern Validation Framework:**
1. **Historical Backtest** → How often did this pattern lead to expected move?
2. **Success Rate by Stock** → Pattern reliability varies by volatility
3. **Market Context** → Pattern strength in bull vs. bear markets
4. **Sector Relative** → Pattern vs. sector index performance

**Output Format for Each Detection:**
```
Pattern Alert:
- Stock: [Symbol]
- Pattern: [Name]
- Timeframe: [Daily/Weekly]
- Detection Confidence: [0-100%]
- Historical Success Rate: [X% over N occurrences]
- Expected Move: [Upside/Downside target]
- Invalidation Level: [Stop loss]
- Key Levels: [Support 1,2 / Resistance 1,2]
- Volume Context: [Confirmation/Warning]
```

### 2.3 Signal Synthesis Agent

**Purpose:** Combine multiple weak signals into strong opportunities

**Signal Categories:**

| Signal Type | Weight | Source | Time Sensitivity |
|-------------|--------|--------|------------------|
| Earnings surprise | 25% | Filings + estimates comparison | Immediate |
| Technical breakout | 20% | Pattern Recognition Agent | Same day |
| Volume anomaly | 15% | Volume vs. historical | Same day |
| Insider activity | 15% | SEBI filings | 24-48 hours |
| News sentiment | 15% | NLP on news flow | Continuous |
| Sector momentum | 10% | Relative strength | Weekly |

**Scoring Algorithm:**
- Each signal gets 0-100 score
- Weighted average = composite opportunity score
- Time decay factor (older signals lose weight)
- Correlation penalty (redundant signals downweighted)

**Opportunity Tiers:**
| Score | Tier | Action | Delivery |
|-------|------|--------|----------|
| 85-100 | **STRONG BUY/SELL** | Immediate alert + detailed report | Push + SMS + Email |
| 70-84 | **MODERATE** | Daily digest highlight | Push notification |
| 55-69 | **WATCH** | Weekly watchlist addition | App notification |
| <55 | **IGNORE** | Log only, no user alert | Internal tracking |

**Confluence Detection:**
- Multiple signals aligning = exponential score boost
- Example: Earnings beat + Breakout + High volume = Score × 1.5

---

### 2.4 Explanation Agent

**Purpose:** Transform raw signals into investor-friendly narratives

**Explanation Framework:**

**Structure for Each Alert:**
1. **The Hook** (1 sentence): What happened and why it matters
2. **The Evidence** (2-3 bullets): Specific data points supporting the call
3. **The Context** (1 paragraph): How this fits broader market/sector trends
4. **The Action** (Clear recommendation): What to consider doing
5. **The Risk** (1 sentence): What could invalidate this signal

**Personalization Dimensions:**
| User Attribute | How to Personalize Explanation |
|----------------|-------------------------------|
| Experience Level | Beginners get more definitions, experts get technical depth |
| Portfolio Holdings | Compare to existing positions, suggest hedges |
| Risk Profile | Conservative users see lower-risk alternatives |
| Past Behavior | Reference previous similar successful alerts |
| Time Horizon | Day traders vs. long-term investors get different targets |

**Tone Guidelines:**
- Confident but not arrogant ("This pattern suggests..." not "This will...")
- Educational ("RSI divergence means...")
- Action-oriented (Specific price levels, not vague "consider buying")
- Honest about limitations ("Success rate is 65%, not guaranteed")

**Multi-Format Output:**
- **Push Notification** (50 chars): "RELIANCE breakout alert +5% target"
- **In-App Card** (200 words): Visual + key stats + quick action
- **Detailed Report** (500+ words): Full analysis, charts, historical context
- **Audio Brief** (60 seconds): For commute listening

---

### 2.5 Portfolio Context Agent

**Purpose:** Filter and prioritize alerts based on user holdings

**User Profile Dimensions:**
- Current holdings (stocks, MFs, ETFs)
- Sector allocation (% in IT, Banking, FMCG, etc.)
- Cash position
- Risk tolerance (Conservative/Moderate/Aggressive)
- Investment horizon (Trading/Swing/Long-term)
- Goals (Wealth preservation/Income/Growth/Speculation)

**Relevance Scoring:**
| Scenario | Relevance Score | Example |
|----------|----------------|---------|
| Alert on held stock | 100 | "Your HDFC Bank position showing breakout" |
| Alert on same sector | 80 | "ICICI Bank breakout - compare to your HDFC" |
| Alert on competitor | 70 | "TCS results beat - impact on your Infosys?" |
| Alert on watchlist | 90 | "Stock you tracked hit target" |
| Alert on unrelated | 30 | Generic market opportunities |

**Smart Filtering Rules:**
- Don't alert on stocks user sold in last 30 days (avoid FOMO)
- Prioritize alerts that fill portfolio gaps (underweight sectors)
- Warn on concentration risk (adding to already large position)
- Suggest position sizing based on volatility

### 2.6 Feedback Loop Agent

**Purpose:** Continuous improvement of signal quality

**Metrics to Track:**
| Metric | Definition | Target |
|--------|------------|--------|
| Alert-to-Action Rate | % of alerts where user took position | >15% |
| Win Rate | % of alerts that reached target | >60% |
| Average Return | Mean return on alert-followed | >5% |
| Time to Target | Median days to reach expected move | <10 days |
| User Satisfaction | Post-alert rating (1-5) | >4.0 |

**Feedback Collection Points:**
- Immediate: "Was this alert useful?" (Yes/No)
- 7-day: "Did you act on this?" + outcome
- 30-day: Full post-mortem on prediction accuracy

**Model Improvement Loop:**
1. Track actual vs. predicted outcomes
2. Identify pattern types with declining accuracy
3. Retrain detection thresholds quarterly
4. A/B test explanation formats
5. Personalize weights per user based on their success patterns
